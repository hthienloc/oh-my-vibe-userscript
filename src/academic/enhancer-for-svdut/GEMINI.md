# Project Guidelines: Enhancer for SVDUT

You are an expert software engineer contributing to the **Enhancer for SVDUT** project. This project is a userscript designed to modernize and add features to the DUT (Da Nang University of Technology) student portals.

## 🛠 Tech Stack
- **Languages:** JavaScript (ES6+), CSS.
- **Runtime/Build:** Bun (used for bundling `src/` into a single userscript file).
- **Environment:** Browser (Userscript manager like Tampermonkey/Violentmonkey).

## 📁 Project Structure
- `src/`: Modular source code.
  - `index.js`: Main entry point and global styles.
  - `lms.js`: Logic for LMS (quiz, auto-answer, diagnosis, summary).
  - `sv.js`: Logic for the student portal (sv.dut.udn.vn).
  - `utils.js`: Shared utility functions (DOM sanitization, image processing).
  - `gemini.js`: Integration with the Gemini AI web interface.
  - `header.js`: Userscript metadata header (Tampermonkey header).
- `build.ts`: Build script that bundles `src/` into the root folder.
- `enhancer-for-svdut.user.js`: The final bundled output (DO NOT edit directly).
- `package.json`: Project metadata and dependencies.

## 🚀 Workflow Rules
1. **Source Control First:** Always modify files in the `src/` directory. Never edit `enhancer-for-svdut.user.js` directly as it is generated by the build process.
2. **Build After Change:** After any modification to files in `src/`, you **MUST** run the build command to update the final userscript:
   ```bash
   npm run build
   ```
3. **VERSIONING (IMPORTANT):** 
   - **DO NOT** manually increment the version in `package.json`.
   - A GitHub Action (`auto-bump.yml`) is configured to automatically bump the patch version, update the metadata header, and commit the changes whenever a push to `main` occurs.
4. **Code Quality:**
   - Adhere to the `Research -> Strategy -> Execution` lifecycle.
   - For bug fixes, reproduce the issue or analyze the DOM via `dom.txt` or provided context before applying the fix.
   - Use `position: sticky` and modern CSS for UI enhancements.
   - Ensure DOM sanitization handles images (convert to base64) to maintain AI context.

## 🧠 Core Features Logic
- **Quiz Diagnosis:** Identifies incorrect questions on `review.php` and prepares a detailed Markdown report for Gemini.
- **Auto-Answer:** Scrapes question text/options, copies to clipboard as JSON prompt, and prompts the user to paste back the result.
- **Summary Load:** On `summary.php`, fetches all quiz attempt pages via `fetch` and injects them into the current view.
- **Sticky UI:** The quiz navigation and critical buttons should remain accessible during long scrolls.
